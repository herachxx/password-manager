Binary Layout

Custom
vault
format

No SQLite. No XML. A tight, versioned binary format designed from scratch. The HMAC is verified before decryption - preventing padding oracle attacks by design.

Encrypt-then-MAC. PassWord refuses to pass a single byte to the AES decryptor until the HMAC is confirmed valid. The order is a security requirement, not a convention.

Capabilities

What's
inside

No black boxes, no external libraries. Every feature is readable, auditable source code you can trace from UI down to raw bytes.

01
AES-256-CBC Encryption
S-box lookups, GF(2⁸) field arithmetic in MixColumns, key schedule expansion, all 14 rounds - readable and verifiable against FIPS 197.
FIPS 197
02
PBKDF2 Key Derivation
600,000 iterations of PBKDF2-HMAC-SHA256. Each wrong guess costs ~0.6s CPU time. Resists GPU acceleration by design.
NIST SP 800-132
03
Encrypt-then-MAC
HMAC verified before decryption. Prevents padding oracle attacks - no bytes reach the AES decryptor until HMAC is valid.
RFC 2104
04
Unbiased Password Gen
Rejection sampling ensures perfectly uniform distribution. The naive modulo bias (~50% skew) is explicitly avoided.
os.urandom()
05
Security Auditor
Scans for password reuse, aging credentials (>90 days), and weak passwords by entropy analysis with actionable warnings.
Built-in
06
SHA-256 Visualizer
Watch every round of the FIPS 180-4 compression function in real time. Three speeds. Verified against Python hashlib every run.
Educational
07
Bruteforce Simulator
Compare SHA-256 vs PBKDF2 crack times for real passwords. Shows exactly why key stretching matters.
Interactive
08
Key Zeroing on Lock
Session keys in mutable bytearray, overwritten byte-by-byte with zeros on lock. Reduces key exposure in memory dumps.
Defense in Depth
09
Zero Dependencies
If it's not in the Python standard library, it's not in this project. No pip, no virtualenv, no external packages - ever.

Clean
layers

crypto.py has zero UI knowledge. vault.py has zero menu knowledge. main.py never touches raw bytes.

This clean separation is why the GUI version (Part 2) can drop in the same core unchanged - zero code duplication between editions.

main.py
Entry point · CLI menus · auth flow · session lifecycle. Never touches raw bytes or cryptographic primitives directly.
UI Layer
tools.py
Password generator · security auditor · bruteforce simulator. No knowledge of encryption or menus.
Tools Layer
vault.py
Binary .vault format · HMAC verification · audit logging. No knowledge of sessions, menus, or UI state.
Storage Layer
constants.py
Constants · custom exception classes · terminal helpers. Shared across all layers with no upward dependencies.
Shared
crypto.py
AES-256-CBC · PBKDF2 · HMAC-SHA256 · SHA-256 visualizer. Pure computation with zero UI knowledge.

Option A - Recommended

Download only install.py. All five source files are embedded as base64 - no pip, no virtualenv, no system packages touched.

python install.pycopy
python install.py ~/apps/my-vaultcopy
python install.py --checkcopy
No recovery possible. Forgotten master password = permanently locked vault. No backdoor, no recovery email. Back up your .vault file regularly.
Option B - Git Clone

Clone and run directly. No build step, no compilation, no setup.py.

git clone github.com/…/password-managercopy
python main.pycopy
python main.py work.vaultcopy
